Measurement Scales

  • Nominal — Domain, gender, state, language tag
  • Ordinal — Serial position within retrieval sequence
  • Interval — SpAM spatial coordinates (planned)
  • Ratio — IRT (ms), word count, education (years)

IRT definition: Time (ms) between successive ENTER key-presses within a domain trial. First word IRT = time from trial start.

Animals

Open-ended; wild + domestic + birds clusters. Largest vocabulary, strongest clustering.

Foods

Richest structure; pulses, meals, snacks. Most culturally specific vocabulary.

Colours

Closed vocabulary (~10 terms). Fastest exhaustion. Highest English proportion.

Body-parts

Internal/external split. Face sub-cluster. Mixed Hindi–English anatomical terms.

What the Experiment Captures

  • VFT phase — Participant types words freely in 1-min window; every ENTER keypress is time-stamped
  • SpAM phase — Participant drag-arranges word tokens on canvas; normalised x, y position recorded per word
  • Practice trial — 1-min Furniture category warm-up; same format as real trials
  • Instruction trails — Hindi-language instructions delivered in-browser; consent implicit via start button

IRT computation: IRTi = response_times[i] − response_times[i−1]. First word resets to time from trial onset.

35 Sessions

One JSON object per participant. Nested structure: session → trials → per-domain arrays.

Time-Stamped

Sub-second precision (100 ms). Enables IRT-level analysis without additional preprocessing.

Bilingual Capture

Devanagari and Latin script both accepted. Language tag applied in post-processing.

Data Engineering Pipeline

From Raw JSON to Analysis-Ready CSV

responses.json ← raw session file; 35 participants, nested VFT + SpAM arrays
   ↓ parse response_times → IRT[i] = t[i] − t[i−1]
   ↓ tag language_type: Devanagari → Hindi, Latin → English
vft_responses.csv ← 35 × 4 domains × N words · cols: subject_id, domain, word, rt_ms, position, language_type
   ↓ filter language_type == "Hindi" → df_hh (712 valid IRTs; 53% of total)
   ↓ cluster detection: switch threshold = mean_IRT + 1 SD (per participant)
   ↓ join with SpAM spatial coordinates on subject_id + domain + word

Language Subsetting

Primary analysis = Hindi tokens only (df_hh, 712 responses). Mixed Hindi+English results shown for context.

Cluster Definition

Troyer et al. (1997) adaptive criterion: switch flagged when IRT exceeds per-participant mean + 1 SD. Consistent cross-participant threshold.

Research Hypotheses

Formal H₀ / H₁ Statements — Before We Look at Data

RQ1 · Within vs Between-Cluster IRT
H₀: µ_WC = µ_BC  (no IRT difference)
H₁: µ_WC < µ_BC  (one-tailed)

Rationale: If words cluster semantically during retrieval (Troyer et al., 1997), pauses between subcategories (between-cluster) should be significantly longer than pauses within a coherent run (within-cluster). Test: Welch's independent-samples t-test.

RQ2 · Cluster Size & Fluency
H₀: ρ(cluster size, fluency) = 0
H₁: ρ > 0  (one-tailed)

Rationale: Participants who sustain longer within-cluster runs have better-organised semantic neighbourhoods and should produce more total words overall. Test: Pearson r with BH correction.

Both hypotheses are directional (one-tailed) — derived from the Clustering-and-Switching Model. We first examine the data exploratorily, then present formal test results & visualisations as the final section.

VFT — Serial Position Effect

Later Words Take Longer: Lexical Exhaustion in Action

Fig 5 · IRT vs Serial Position — per-domain scatter with OLS trend lines and 95% CI bands

Each dot = one word produced by one participant at a given ordinal rank (1st word, 2nd word, …). OLS trend lines are fitted independently per domain. All four lines slope upward — later-position words always cost more retrieval time. Colours has the steepest slope: by position ~10, most Hindi colour terms are exhausted, so each additional word demands genuine search effort. Foods has the shallowest slope: the large, hierarchically structured vocabulary (pulses → meals → snacks → condiments) keeps the semantic neighbourhood well-populated for longer.

What is Serial Position?

The ordinal rank of a word within a single 60-s trial. Word 1 = first word typed, word N = last. Captures how retrieval speed changes as the word list grows longer.

Lexical Exhaustion

As words are retrieved, the local semantic neighbourhood depletes — like foraging a patch until it is stripped. The participant must either switch subcategory (cluster switch → large IRT) or slow down while searching the same depleted patch.

What the result means

A Pearson correlation of r = .57 (p = .005, one-tailed) was found between mean cluster size and total Hindi words produced per participant. The 95% CI [0.31, 0.75] excludes zero, confirming a significant positive relationship. Cluster size explains 33% of the variance in fluency (r² = 0.33).

Regression equation

Total words = 3.12 + 2.14 × Mean cluster size. Each additional unit of cluster size predicts approximately 2 extra words produced within the 1-minute trial. Participants who sustain longer in-cluster runs — indicating better-organised semantic sub-networks — retrieve more words overall.

Interpretation

Clustering efficiency is a strong predictor of verbal fluency in Hindi. Participants with richer, more densely connected semantic sub-networks can stay within a subcategory longer before switching, thereby producing more total responses in the fixed time window.
